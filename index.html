Payment Flow & Edge Cases
Implementation reference — payment lifecycle, state machine, and edge case handling

Subscription State Machine

Incomplete
→
Active
⇄
Paused
→
Cancelled
Active ↔ Paused (manual or auto from failed payment) · Cancelled is terminal · Past Due = Active with flag

Payment / Dunning Flow

Subscribe → PayTheory tokenize → Store token → Pre-charge email (3 days before) → Charge via PayTheory → Success / Fail
If fail: Day 3: Retry 1 → Day 6: Retry 2 → Day 9: Retry 3 → Auto-pause + email both parties
3-retry matches industry standard (Stripe, ReCharge). Auto-pause is better than auto-cancel — keeps the relationship alive. Hard declines (stolen card, closed account) should NOT be retried. Only retry soft declines (insufficient funds, temporary holds).

Key Design Decisions for David

1. Subscriptions nav item under SELL — between Products and Inventory 2. Farm Box contents set per-week on SubscriptionOrder.box_contents, not on the Plan 3. PayTheory token stored once at subscribe, reused for all recurring charges 4. 3-retry dunning (day 3, 6, 9) then auto-pause (NOT auto-cancel) 5. No proration in V1 — all changes take effect at next billing cycle 6. Capacity limits are hard — when full, button changes to "Join Waitlist" 7. Blackout dates skip the charge entirely — no defer, no reschedule 8. Cancel flow max 3 screens — FTC compliant, each is an optional off-ramp 9. Subscribe & Save links to existing products — no product duplication needed
